# 90-Day Buyer Operating Plan

## Days 0–30: establish proof and supportability

- Transfer repository and Vercel ownership; preserve preview protection defaults until a buyer demo is planned.
- Run the transfer checklist and publish a dated verification record.
- Capture approved desktop/mobile demo media after anonymous preview access is explicitly enabled.
- Interview target consultants/agencies against the three shipped workflow examples; do not claim traction before it exists.

## Days 31–60: validate the commercial wedge

- Test landing-page and template messaging around discovery notes, meeting action plans, and research briefs.
- Add buyer-approved template refinements from observed workflows.
- Decide whether cloud sync is justified; if yes, design authentication, authorization, encryption, retention, and recovery before implementation.

## Days 61–90: choose the scale path

- Prioritize one of: team collaboration, secure AI proxy, or verticalized template packs.
- If monetization begins, add billing and account controls as a separate audited product surface.
- Keep local-first export/recovery intact as a product promise while new services are introduced.
